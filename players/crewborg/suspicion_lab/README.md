# suspicion_lab — fitting crewborg's suspicion model from scraped replays

The **data-science half** of crewborg's suspicion system: scrape league games, expand their replays
into observer-exact evidence, fit evidence weights against ground-truth roles, and emit a weights
file the agent loads. It produces **[`crewborg/data/suspicion_weights.json`](../crewborg/data/suspicion_weights.json)**
— the artifact the runtime scorer ([`crewborg/strategy/suspicion.py`](../crewborg/strategy/suspicion.py),
documented in [`crewborg/docs/suspicion.md`](../crewborg/docs/suspicion.md)) loads.

> **Status: prototype.** It works end-to-end and produced the shipped weights, but it's research-grade
> — expect rough edges, and read the code before trusting a number.

> **⚠️ There is NO data in this folder.** `corpus/`, `expanded/`, `dataset/`, `models/`, and `logs/`
> are **empty in a fresh checkout** — they are pipeline *outputs*, gitignored, and **must be
> generated by running the pipeline below**. The corpus alone is ~50 GB, so it is never vendored.
> **crewborg already ships a fitted `crewborg/data/suspicion_weights.json`, so the player works
> without any of this** — you only come here to *refit / improve* the weights, which means first
> rebuilding the corpus from scratch.

## Pipeline (each stage idempotent; re-run any time)

Run from the player root (`players/crewborg`). Stages A→D fill the (initially empty) data dirs.

```sh
# A. scrape completed league rounds (replay + results per episode) into suspicion_lab/corpus/
uv run python suspicion_lab/tools/scrape_corpus.py --max-rounds 12

# B. expand replays -> suspicion_lab/expanded/<episode>.jsonl.gz. Needs a version-matched
#    expand_replay binary at tools/bin/expand_replay-<ref> — build it first:
#       tools/build_expand_replay.sh --ref 42fed21      # use the deployed game's ref
uv run python suspicion_lab/tools/expand_corpus.py --workers 8

# C. per-(observer, suspect, meeting) feature rows + labels -> suspicion_lab/dataset/dataset.parquet
uv run python suspicion_lab/tools/build_dataset.py

# D. fit weights (L1 logistic regression, game-grouped CV) -> suspicion_lab/models/<tag>/
uv run python suspicion_lab/tools/fit.py --tag <tag> --features runtime
#    --features runtime = only what crewborg's live event log can compute (SHIP THIS)
#    --features full    = the research ceiling (every offline feature)

# E. decision-level eval: vote policies vs the always-skip baseline
uv run python suspicion_lab/tools/eval.py --model suspicion_lab/models/<tag>
```

To ship a refit: copy the chosen `models/<tag>/suspicion_weights.json` to
`crewborg/data/suspicion_weights.json`, then build + Gate-1 + A/B + (gated) submit — the
`build-and-upload` and `coworld-policy-lifecycle` skills. (The `nightly_refit.sh` loop does all of
this unattended; see below.)

## Files (all under `tools/`)

- **`scrape_corpus.py`** — round-ledgered incremental scraper (wraps the `coworld-episode-artifacts`
  fetcher). A per-round ledger (`corpus/_rounds_done.json`) makes it resumable.
- **`expand_corpus.py`** — runs the version-matched `expand_replay --format jsonl` over the corpus;
  `_manifest.json` records per-episode status (`hash_failed` = recorded on a different game build — the
  version-coupling gotcha, see `tools/build_expand_replay.sh`).
- **`replay_parse.py`** — expanded JSONL → a typed `Game` (players/roles, sampled states, visibility
  intervals, kills/bodies/ejections, meetings, chats). NB: roles come from `player_state` rows, not
  the join-time `player_manifest`.
- **`features.py`** — the evidence catalogue: cumulative, visibility-clipped, instance-summed
  per-(observer, suspect) features + chat-stance triples. **Every feature must stay
  runtime-admissible** (computable from crewborg's live event log).
- **`build_dataset.py`** — the feature rows + a per-cue imposter/crew sanity report.
- **`fit.py`** — the model + the transform contract (`BIN_SPEC`, `LINEAR_CLIP`) that ships inside
  `suspicion_weights.json` and is mirrored by the runtime scorer.
- **`eval.py`** — held-out (out-of-fold) meeting decisions through a vote-policy grid; ship only what
  beats always-skip on net parity.
- **`nightly_refit.sh`** — the unattended cron loop (below).

## How to change it

- **Add an evidence cue** → add a runtime-admissible feature in `features.py`, rerun C→E. (If it isn't
  computable from crewborg's live events, it can't ship — keep `--features runtime` honest.)
- **Change the model / regularization / CV** → `fit.py`. Keep the `BIN_SPEC`/`LINEAR_CLIP` transform
  contract in sync with the runtime scorer (`crewborg/strategy/suspicion.py`), or the weights won't
  mean the same thing live.
- **Change vote policies / the ship bar** → `eval.py`.

## The nightly champion loop

`tools/nightly_refit.sh` runs the whole pipeline unattended (a user crontab entry, `30 0 * * *`):
scrape → expand → dataset → fit → **gates** (CV AUC ≥ 0.70, corpus ≥ 500 games, the full crewborg
test suite, a local Gate-1 smoke — any failure aborts with the current champion untouched) → vendor
the weights → build → upload → **submit**. The auto-submit is the *lab's* standing operational choice
(it leans on the gates); **adapt it for your own use** — submitting is the gated, irreversible action.
`--check` verifies prerequisites (uv, docker daemon, softmax auth). Paths are derived from the script
location, so it runs wherever the repo is checked out.

## What the lab achieved (for reference — not what's in this folder)

The last full-corpus fit reached **runtime-model AUC ≈ 0.81** on ~1,857 games / ~196k rows, with the
held-out decision sim hitting ~94% imposter precision at P≥0.9 (net positive vs always-skip).
`tasks_completed_watched` was the single strongest cue (imposters produced ~zero across the labelled
rows). **That corpus is not shipped** — these numbers describe a past run; reproduce them by
regenerating the corpus and refitting.
